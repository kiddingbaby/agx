# v0.1.0

> 初始基线发布。 · English: [v0.1.0.md](v0.1.0.md)

首个打 tag 的版本。把 agx 确立为单用户 CLI：管理 OpenAI 兼容的中转
profile，并把 profile 同步给 `codex` / `claude` / `gemini` /
`opencode`，每个 agent 跑在隔离的受管上下文里。

## 表面

- **Profile 生命周期** —— `agx add` / `edit` / `rm` / `ls` / `show` /
  `use` / `current`。`base_url` + `api_key` 必填；`model` 可选（启动
  `opencode` 时必填）。
- **Launcher** —— `agx run <agent> [profile] [-- <native args>]` 以及
  按 agent 的别名 `agx codex` / `claude` / `gemini` / `opencode`。每个
  agent 跑在 `~/.config/agx/contexts/<agent>/<name>/`，不动宿主机的
  `~/.codex` / `~/.claude` / `~/.gemini` / `~/.config/opencode`。
- **Profile 解析优先级** —— 位置参数 `profile` > `AGX_PROFILE` 环境
  变量 > `agx current`。可配合 `direnv` 按目录 pin，不动全局默认。
- **Backup / restore** —— `agx backup <agent>` 给当前 managed target
  拍快照；`agx restore <agent>` 回滚最近一次快照（支持 `codex` /
  `claude` / `gemini`）。
- **Opt-in 自动备份** —— `AGX_AUTO_BACKUP=1` 时 `agx run` 启动前自动
  snapshot。
- **诊断** —— `agx doctor` 列出问题并附建议动作；每个能 emit 的
  `code` 都在 [`docs/doctor-issues.zh.md`](../doctor-issues.zh.md) 里
  有 severity / 含义 / 推荐修复，测试锁住文档与运行时 emitter 的
  双向一致。
- **Mutation guard** —— 每个修改类入口（add/edit/remove、managed
  profile 流、target API、binding helper、backup/restore）写盘前都
  capture profile YAML、state、各 agent 磁盘配置；任一步失败整体回滚；
  panic 也跑完 rollback 再重新 panic 保持非零退出。
- **Shell completion** —— `agx completion {bash|zsh|fish|powershell}`。
- **版本信息** —— `agx version` 输出 `version` / `commit` / `date` /
  `go` / `os` / `arch`。
- **First-run UX** —— 空 store 时 `agx` 和 `agx ls` 给引导。
- **Man pages** —— 19 个 man page 随 release 发布，Homebrew formula
  自动装。

## 安装

- **Homebrew tap** —— `brew install kiddingbaby/agx/agx`。
- **预编译二进制** —— Linux / macOS，amd64 / arm64。
- **`go install`** —— `go install github.com/kiddingbaby/agx/cmd/agx@latest`。
- **源码构建** —— Go 1.24+，工具链由 `mise.toml` 声明。

## 稳定性

Pre-1.0。公开 CLI 表面见 [`docs/cli-contract.zh.md`](../cli-contract.zh.md)，

字段、文件布局可能在 1.0 前调整，请勿依赖。
